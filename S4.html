  • Ensure the internal office is running smoothly on the IFC platform.
  • Achieve minimal bugs and a positive user experience for all internal users.
  • Complete and refine features from Flow 1.0 / P3 to ensure a cohesive platform.
  • Lay the groundwork for future sales enablement and partner onboarding epics.

‚ú® Key Features & Enhancements for Sprint S4

1. Header Redesign & Consistent Information Display IMPROVEMENT

Description: Redesign the project view header to consistently display primary information. This includes:

  • Client details (name, contact, address).
  • Sales representative.
  • Insurance information (Claim #, Insurer, Adjuster details).
The current blue pill buttons (CONVO, FLOW, STATUS) will be replaced by a standard tab system for navigation within a project.
Impact: Improves information accessibility, reduces clicks, and standardizes UI across desktop and mobile.

2. Task Management: Checklists (Subtasks) NEW

Description: Implement "Checklists" within Tasks.

  • A task can contain multiple checklists.
  • Completion of all checklist items will signify task completion.
  • Display percentage of task completed based on checklist items.

Impact: Critical for accountability (e.g., Vanessa's process for SOT/bid verification) and breaking down complex tasks into manageable steps. This is a priority to address current process gaps.

3. Construction Plan UI & Logic Refinements IMPROVEMENT

Description: Implement several key improvements to the existing construction plan / roof details form:

  • Field Reordering: Reorganize fields for better chronological data entry (e.g., shingle style/color at the top), as per Michel Browder's template.
  • Binary "Yes/No" for Added Items: Change the "added" indicator for supplemented items to a simple "Yes" or "No" (instead of "1+").
  • Shingle Input Refinement: Add a "Class 4 (Yes/No)" option. (Future: More structured input with manufacturer, style, class 4, color dropdowns).
  • New Fields: Add "HVAC" and "Personal Property" (for noting damaged items) fields.
  • Garage Door Options: Add a dropdown with "Replace" or "Paint/Stain".
  • Gutters Options: Add options like "Remove gutters prior to roof" or "Remove at install".
  • Skylights: Add "Skylights (Yes/No)" option.
  • Roofing Boot Terminology: Change "Rubber" to "Three in One". Add note for "Bullet Boots" (suitable for pitches 9 and below).
  • Chimney Field: Rename to "Flashing" with yes/no options for chimney and skylights.

Impact: Improves clarity, data accuracy, and efficiency for the production and supplementing teams. Reduces scrolling and makes the form more intuitive.

4. "Thumbs Up" Logic in Construction Plan FIX

Description: The default for the "Thumbs Up/Down" (indicating if work is being done) will be set to "Thumbs Down". It will only change to "Thumbs Up" when work is explicitly confirmed.
Impact: Prevents confusion where work appears to be approved for execution based only on funding, ensuring clarity on actual scope.

5. Board Filtering Enhancements IMPROVEMENT

Description: Add filtering capabilities to board views (e.g., Active Jobs).

  • Filter by Sales Representative.
  • Display "Last Client Contact Date" on cards.
  • Make filters "sticky" (persist across sessions or navigation).

Impact: Critical for customer service and sales management to track progress and engagement effectively.

6. Notion API Integration - First Action: Warranty Inspection Protocol NEW

Description: Implement the first Notion API integration. Triggering a "Warranty Inspection" from the FLOW app will:

  • Create a calendar invite (Google Calendar).
  • Create related pages/links in Notion for the inspection protocol.

Impact: Streamlines the warranty inspection process and serves as a pilot for further Notion integrations (e.g., material order forms).
